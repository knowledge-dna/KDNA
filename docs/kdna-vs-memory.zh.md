# KDNA 与记忆系统

AI 记忆系统记住发生了什么。KDNA 解释它意味着什么。

## 核心区别

**记忆问：** 之前发生了什么？用户偏好什么？我们上次做到哪了？

**KDNA 问：** 在这个领域里，这种行为模式实际上意味着什么？

## 五层架构

| 层 | 角色 | 示例 |
|----|------|------|
| 模型 | 推理和生成 | GPT, Claude |
| 知识 | 事实和文档 | RAG, 数据库, 网络 |
| 记忆 | 历史和连续性 | 用户历史, 偏好, 会话上下文 |
| **KDNA** | **领域判断** | **公理, 误判, 信号, 自检** |
| 行动 | 执行 | MCP, 工具, 工作流 |

KDNA 位于记忆和行动之间。记忆提供关于发生了什么的上下文。KDNA 在该背景下，在 Agent 行动之前，提供关于上下文的判断。

## 为什么这很重要

没有判断的记忆是危险的。一个 Agent 记得客户问过三次价格，但不知道反复问价意味着确定性不足——就会不断给折扣。一个 Agent 记得员工错过了截止日期，但不知道反复错过意味着系统上游问题——就会一直归咎于动力。

记忆记录信号。KDNA 解释信号。

## 对比

| | 记忆 | KDNA |
|---|------|------|
| 核心问题 | 发生了什么？ | 在这个领域意味着什么？ |
| 存储内容 | 事件、偏好、历史 | 公理、误判、信号、自检 |
| 输出 | 可检索的上下文 | 可加载的判断结构 |
| 价值 | 连续性、个性化 | 专业判断、防误判 |
| 风险 | 过度记忆、隐私 | 模糊规则、错误域应用 |

## 它们一起工作

> 记忆告诉 AI 发生了什么。KDNA 告诉 AI 这意味着什么。Skills 告诉 AI 下一步该做什么。

客服 Agent：记忆调取客户的投诉历史。KDNA 判断这个模式是服务故障、期望落差还是风险升级。Skills 执行对应的响应。

KDNA 不是记忆系统。它是与任何记忆系统兼容的判断层——为已记住的上下文增加领域解释。
