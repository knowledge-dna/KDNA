# KDNA 安全框架

## 设计原则

1. **判断 ≠ 授权**：加载 KDNA 不会授予 Agent 新权限
2. **Human Lock 作为安全门**：每条判断卡片必须经人类逐条确认
3. **质量门防止低质量判断**：模糊公理、缺失边界、稻草人误解会被拦截
4. **风险等级声明是强制的**：每个领域必须声明 R0-R3 风险等级
5. **来源追踪是必需的**：未签名的 .kdna 不可信

## 高风险领域政策

医疗诊断、法律建议、投资建议、心理健康危机、儿童安全、监控系统、武器相关、政治操纵等领域默认 R2-R3，进入公共注册表需特殊审核。

## 安全加载规则

1. 检查风险等级 — 高风险领域不应静默加载
2. 检查签名 — 未签名领域应触发警告
3. 检查 yank 状态 — 已 yank 领域必须拒绝
4. 尊重优先级 — 系统安全 > 用户意图 > KDNA 判断
5. 记录加载 — 记录加载了什么、为什么、触发了什么
