# 商业化路径

KDNA 将开源协议和商业资产分层。以下是什么是免费的，什么是付费的，以及为什么。

## 什么是开源的

| 组件 | 许可 | 原因 |
|------|------|------|
| KDNA 格式规范 | Apache 2.0 | 标准必须免费采用 |
| JSON Schema | Apache 2.0 | 任何人都能验证 KDNA 文件 |
| 验证器 | Apache 2.0 | 任何人都能检查领域质量 |
| CLI | Apache 2.0 | 任何人都能安装使用 |
| Loader 库 | Apache 2.0 | 任何人都能集成到 Agent |
| Skills | Apache 2.0 | 任何人都能在 Agent 中加载 KDNA |
| Basic 与实验领域 | CC BY 4.0 | 降低使用门槛 |
| Benchmark 案例 | CC BY 4.0 | 社区可以贡献与验证 |

## 什么是可以商业化的

| 组件 | 模式 | 原因 |
|------|------|------|
| Pro 领域（写作、演讲等） | 付费许可 (KCL) | 专家判断力是资产 |
| KDNA Runtime | 付费 / 托管 | 任务投影、许可、水印 |
| 企业私有 KDNA | 咨询 + 许可 | 编码内部专业知识 |
| Creator 认证 | 认证费 | 对买家的质量保证 |
| 托管 Registry | 订阅 | 发现、版本管理、分析 |

## 为什么这样分层

协议必须开源，KDNA 才能成为标准——像 HTML、Markdown 或 npm 一样。如果格式是专有的，没人会基于它构建。

资产可以商业化，因为价值不在格式——而在专家的判断结构。这是开源生态一直以来的运作方式：平台免费，内容可商业化。

## 当前状态

我们处于 Phase 4：构建 KDNA 改善 Agent 判断的证据。商业资产（Pro 领域、Runtime）存在于私有仓库，将在证据足够强时发布。
